Stock Bench

Read-only market view for a pool-relative, council-scored LLM market.

How this dashboard works

Stock Bench runs a repeated market loop between models. In each round, the system picks a task, asks every model to bid on how well it thinks it will do, selects one model to answer, then asks the other models to score that answer.

The dashboard shows the running state of that process. Market standings summarize each model's current price, uncertainty, and calibration. Recent rounds show the latest task cycles, including who answered, how well they scored, and how accurate their self-estimate was.

These numbers are pool-relative: they help compare models inside this current test pool, not across completely different pools. Early rounds are also marked as provisional because the system is still learning.

Tip: hover over column names, round labels, metric chips, and task text for more explanation.
